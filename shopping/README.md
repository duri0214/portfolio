# shopping - 出店計画と購入デモ

## 概要

`shopping` は、商品購入デモと出店計画の分析画面を持つ Django アプリケーションです。

出店計画機能では、店舗候補の座標や e-Stat 人口統計を使い、候補地の周辺地域を比較します。将来的には Google Maps のレビューや Places API 由来の指標も組み合わせ、出店判断の補助材料を増やします。

## 出店計画の位置づけ

出店計画は、最終的な出店可否を自動判定する機能ではありません。

以下のような判断材料を同じ画面で確認し、候補地を比較しやすくすることを目的にします。

- 周辺町丁の人口構成
- 店舗候補の位置と周辺地域
- 口コミやレビューから見えるエリアの評判
- 周辺施設密度などから推定する賑わい

## Google Maps レビュー拡張の考え方

Issue #131 では、Google Maps レビューを単一店舗の評価として扱うのではなく、出店候補地周辺の評判を広く聞くためのデータとして扱います。

### やりたいこと

- 対象地点から半径 500m 程度を初期分析範囲にする
- 範囲を 3x3 の簡易グリッドに分割する
- 取得済みの店舗や施設レビューをグリッド単位で集約する
- レビュー本文からポジティブ傾向、ネガティブ傾向、よく出る話題を抽出する
- グリッドごとのスコアを CSS Grid のヒートマップとして表示する

### 初期実装で避けること

- Google Maps 上へ直接 HeatmapLayer や Polygon を描画する
- 境界ポリゴンを厳密に扱う GIS 実装にする
- スコアの精度を過度に作り込む
- LLM 要約、ベクトル検索、DB正規化を一度に実装する

まずは、ユーザーが「この候補地の周辺では、どのあたりに良い評判や悪い評判が偏っているか」を比較できる状態を優先します。

## データ設計の方向性

Google Maps レビュー拡張では、以下の役割に分けて段階的に実装します。

| 領域 | 役割 |
| --- | --- |
| MySQL | 店舗、施設、レビュー、グリッド集計結果を正規化して保存する |
| Chroma | レビュー本文や要約を検索・分類しやすい形で保持する |
| Domain Service | レビュー集約、簡易スコア計算、グリッド分割を担当する |
| Template | 出店計画画面で、人口統計や地図と並べて評判ヒートマップを表示する |

MySQL と Chroma は併用候補ですが、MVPでは MySQL に保存済みの集計結果を表示するところから始めます。Chroma は、レビュー本文の横断検索や類似トピック抽出が必要になった段階で追加します。

## 現在の出店計画画面

`/shopping/store-planning/` では、以下を表示します。

- 店舗候補の選択
- 対象地点と人口集計地域
- Google Maps iframe による地域表示
- e-Stat CSV 由来の周辺地域比較
- e-Stat 年代別人口と男女構成
- CSV カバー範囲

レビュー拡張は、この画面の分析軸として追加する想定です。
