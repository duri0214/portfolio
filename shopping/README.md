# shopping - 購入デモ・出店計画

## 概要

`shopping` は、商品購入デモと出店計画画面を持つ Django アプリケーションです。

商品購入では Stripe の Payment Intents API を使った決済確認フローを扱います。出店計画では、店舗候補の座標を起点に、周辺地域の人口構成、地図、保存済み Google Maps レビューを同じ画面で確認できます。

## 主な機能

- 商品一覧、商品詳細、商品登録
- CSV による商品一括登録
- Stripe 決済確認と購入履歴保存
- 非ログインユーザーの購入制限
- 出店計画の店舗候補登録
- 出店計画の対象店舗切り替え
- e-Stat CSV 由来の年代別人口表示
- e-Stat 地域コードによる周辺地域比較
- Google Maps iframe による地域表示
- 店舗候補に紐づく Google Maps レビュー表示
- スーパーユーザーによる Google Maps レビュー取得

## 出店計画画面

`/shopping/store-planning/` では、登録済みの店舗候補を選択して、出店判断に使う情報を確認します。

表示している内容は以下です。

- 対象地点
- 人口集計地域
- Google Maps レビュー集約
- 地域マップ
- 周辺地域比較
- e-Stat 年代別人口
- e-Stat CSV カバー範囲

## Google Maps レビュー

出店計画画面では、`StorePlanningGoogleMapsReview` に保存したレビューを店舗候補ごとに表示します。

レビュー欄ではレビュー件数、平均 rating、ポジティブ・ネガティブ語の件数、簡易スコア付きの3x3グリッド、代表レビューを表示します。

出店計画画面の表示時に外部リクエストは送りません。保存済みレビューがない場合は、ログイン済みスーパーユーザーだけが画面上の「レビュー取得」ボタンからレビューを取得できます。取得済みレビューは店舗キーごとに保存し、同じ店舗キーのレビューが保存済みであれば再取得しません。

## 店舗候補

店舗候補は `StorePlanningTargetStore` モデルで管理します。

店舗候補には以下を保存します。

- 店舗キー
- 店舗名
- 住所
- 緯度・経度
- e-Stat 市区町村コード
- e-Stat 町丁字コード
- 人口集計地域
- 大字・町名
- 字・丁目名
- 表示対象フラグ

## 関連コマンド

e-Stat の出店計画データは以下の管理コマンドで取得します。

```bash
python manage.py daily_fetch_store_planning_data_sources
```
