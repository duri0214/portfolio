# Soil Analysis アプリケーション

土壌分析データの管理、および気象庁データに基づく予報管理を行うアプリケーションです。

## ドメイン知識

### 圃場ブロック (LandBlock)

圃場は 3x3 の 9ブロックに分割して管理されます。
土壌硬度（物理性）の計測では、このブロック単位でデータが保持されます。

#### ブロック配置図

```text
┌────┬────┬────┐
│ C3 │ B3 │ A3 │  (Row 3)
├────┼────┼────┤
│ C2 │ B2 │ A2 │  (Row 2)
├────┼────┼────┤
│ C1 │ B1 │ A1 │  (Row 1)
└────┴────┴────┘
 (Col C) (Col B) (Col A)
```

- **行 (Row):** 1, 2, 3
- **列 (Col):** A, B, C
- **計測シナリオ:** 5点計測（C1, C3, B2, A1, A3）などのパターンがあります。

### データ構造の責務分離

#### 物理性データ (`SoilHardnessMeasurement`)

- **単位:** ブロックごと (`LandBlock`)
- **内容:** 深度ごとの貫入抵抗値（圧力）。
- **計測（貫入）の仕様:**
    - **計測点数:** 1つのブロックに対して複数箇所（標準的には5箇所または9箇所）の測定ポイントで貫入計測を行います。
    - **平均化:** 解析・表示時には、これら複数箇所の測定値を深度ごとに平均化して扱います。
- **管理:** 1つの計測（貫入）に対して、複数の深度レコードが紐づきます。

#### 化学性データ (`SoilChemicalMeasurement`)

- **単位:** 圃場（台帳）ごと (`LandLedger`)
- **内容:** EC, pH, 窒素、リン酸、カリウム、塩基置換容量(CEC)などの分析値。
- **背景:** 以前はブロックごとに複製保持していましたが、データ冗長性を排除し、圃場単位の 1:1 関係 (`OneToOneField`)
  に集約されました。

### モデル間の関係

- **Land (圃場):** 地理的な場所、面積、オーナー情報を保持。
- **LandPeriod (時期):** 「2024年 春」のような、分析の実施タイミングを管理。
- **LandLedger (台帳):** `Land` と `LandPeriod` の組み合わせに対して、分析依頼日、報告日、分析機関、作物などを紐づける中心的なモデル。
- **SoilChemicalMeasurement:** `LandLedger` に 1:1 で紐づく化学分析結果。
- **SoilHardnessMeasurement:** `LandLedger` と `LandBlock` に紐づく物理性計測データ。

## 土壌物理性診断 測定作業マニュアル (要約)

### 1. 目的

次作の施肥設計を調整するために、土壌の深さに対する硬さの推移（土壌硬度）を測定する。基本的には採土作業とセットで実施する。

### 2. 作業フロー

1. 圃場情報の収集（位置、時期、品目、施肥計画、目標収量など）
2. 現地での硬度測定
3. 測定データのアップロード（専用アプリを使用）
4. 測定データに圃場・測定情報を紐付け（マクロを使用しCSV/Excel統合）

### 3. 測定のルール

- **測定時期:** 前作の収穫・片付けが終了し、荒耕起の状態で、かつ次作の施肥前（施肥の4週間以上前）が理想。
- **測定位置 (9分割/5分割):**
    - 圃場入口を手前として基準を設ける。
    - 9分割の場合、A1〜C3の全9エリアを測定。
    - 5分割の場合、A1, A3, B2, C1, C3 の5エリアを測定（採土位置と同じ）。
- **地点内の測定:**
    - 各エリア（ブロック）内で指定された回数（5回または9回など）測定する。
    - 1圃場（5エリアの場合）あたり計25ポイント以上を測定することになる。
    - 畝がある場合は、根が生える畝の内部を少しずつずらして測定する。
    - サブソイラ等を使用している場合は、溝の影響を確認するため斜めに測定する等の工夫を行う。

### 4. 機器設定（初期状態）

- **使用機器:** 土壌硬度計 (DIK-5531/5532)
- **設定値:** メモリ番号400、コーン2cm、設定深度60cm

## 土壌化学性診断 採土作業マニュアル (要約)

### 1. 目的

次作の施肥設計を調整するために、土壌の化学成分（養分状態）を分析するための検体（土壌）を採集する。

### 2. 作業フロー

1. 圃場情報の収集（位置、時期、品目、施肥計画、目標収量など）
2. 現地での採土
3. 採土した土の風乾（日陰で乾燥させる）
4. 検査機関（株式会社川田研究所）に検体を送付
5. 検査結果を圃場情報と紐付け

### 3. 採土のルール

- **採土時期:** 前作の収穫・片付けが終了し、荒耕起の状態で、かつ次作の施肥前（検査結果が出るまで2〜3週間かかるため、施肥の4週間以上前）が理想。
- **採土位置:**
    - 1圃場を単位とし、代表的な複数箇所（標準的には5箇所など）から採土する。
    - 畦畔、道路、水路などから10m程度離れた場所を選ぶ。
- **採土の手順と深さ:**
    - 表面の有機物（残渣）を除き、作土（およそ 0〜15cm）を採土する。
    - 平坦地（畝なし）: スコップで穴を掘り、移植ゴテで斜め柱状に採土する。
    - 作畝地（畝あり）: 株間で畝肩から次の畝肩までの作土を、畝方向に垂直に採土する。
- **量と混合:**
    - 採取した複数箇所の土（各150cc程度）をバケツ等でよく混ぜ合わせ、その中から500g程度を検体袋に入れる。

### 4. 風乾と送付

- **風乾:**
    - 容器に広げ、礫や根、残っている肥料粒等を取り除く。
    - 直射日光を避け、風通しの良い日陰で扇風機等を使って「白乾」するまで乾燥させる。
- **送付:**
    - 十分に乾燥した土を約300g、送付用ジップロック袋に詰める。
    - 検査機関（川田研究所）へ宅配便で送付する。

## 主要なバッチ・コマンド

### インポート

- `ChemicalImportService`: 川田研究所形式の Excel ファイルから化学分析データを取り込みます。
- `HardnessImportService`: 土壌硬度計の CSV データを取り込みます。

### テストデータ生成

- `hardness_generate_dummy_csv`: 土壌硬度計測のダミーデータを生成します。
