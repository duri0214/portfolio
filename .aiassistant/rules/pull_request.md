# プルリクエスト (PR) 作成規約

PRを作成する際は、以下の構成で記述すること。

## 1. 修正概要
- 何を変更したのか、なぜ変更したのかを簡潔に記述する。
- 関連するIssue番号があれば記載する。

## 2. 自動テストでカバーできた範囲
- どのテストファイルを実行したか。
- 新規追加したテストケースの内容。
- 既存テストへの影響（すべてパスしたことの確認）。

## 3. 動作確認チェックリスト
- 画面上で目視確認、または手動で操作して点検すべき項目を箇条書きで記載する。
- 正常系だけでなく、想定される異常系の挙動も含むこと。

## 5. 提示方法
- プルリクエストの内容を依頼された場合は、常にこの規約に基づいたMarkdown形式のテキストとして出力すること。
- 直接 `submit` するのではなく、回答や提案の中に含めて提示すること。

---

### テンプレート例

```markdown
### 修正概要
- [内容を記述]

### 自動テストでカバーできた範囲
- [テストファイル名] を実行
- [新規テストの内容]
- 全ての既存テストがパスすることを確認

### 動作確認チェックリスト
- [ ] [確認項目1]
- [ ] [確認項目2]
```
