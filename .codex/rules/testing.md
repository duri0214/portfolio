# テストコード規約

テストコードを記述する際は、後で見返した時に意図が明確に伝わるよう、以下のルールを遵守する。

## Docstring への「シナリオ」の記述

すべてのテスト関数（メソッド）には、そのテストが何を検証しているのかを説明する docstring を記述する。
docstring には以下の「シナリオ」を含めること。

- **入力**: どのようなデータ（引数、モックの戻り値、DBの状態など）を与えるか。
- **処理**: どのような操作（メソッド呼び出し、関数の実行など）を行うか。
- **期待値**: どのような結果（戻り値、副作用、例外の発生など）が返されるべきか。

### 記述例

```python
def test_calculate_total_score():
    """
    シナリオ:
    - 入力: 正確性: 3, 論理: 4, 独創性: 2, 反論: 0 の評価データ。
    - 処理: total_score プロパティを呼び出す。
    - 期待値: 合計値 9 が返されること。
    """
    # ... 実装
```

## テストの構成 (Given-When-Then)

docstring のシナリオに沿って、コード内でも Given (前提条件), When (実行), Then (検証) の構成を意識して記述する。
