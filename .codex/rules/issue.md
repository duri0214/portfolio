# Issue 作成規約

- **出力言語は必ず日本語とすること。**
- ユーザーから「issue書いて」「リファクタリングチケット作って」「チケットを書いて」等の Issue
  作成の依頼があった場合は、以下の「生成フォーマット」に基づき、内容を **Markdown のコードブロック（生md）** で生成すること。

## 生成フォーマット

Issue を生成する際は、**必ず以下の Markdown コードブロック（三重バッククォート）で囲んだ状態で記述すること。**

```markdown
**タイトル:** [ここにIssueのタイトルを記述]

**本文:**
## 概要
[ここに対応内容の概要を記述してください]

## 主な対応内容
- [具体的な対応項目1]
- [具体的な対応項目2]

## 期待される効果
- [この対応によって得られるメリット1]
- [この対応によって得られるメリット2]
```
