# 外部データ表示の透明性

e-Stat、EDINET、Yahoo API、官公庁統計、CSV取り込みなど、外部データを画面・レポート・ダッシュボードに表示するときだけ読む。

- 外部データを画面・レポート・ダッシュボードに表示する場合は、ユーザーがその値の信頼性を判断できる情報を近くに表示する。
- 少なくとも、取得元、データ時点または対象期間、ローカルDBへ保存した日時（アプリ側の取得日時）を表示する。
- 外部データに公表日・更新日などのメタ情報が含まれる場合は、それが「保存済みデータに含まれる提供元公表/更新日」であることが分かるラベルで表示する。提供元の現在最新日時と混同しない。
- 取得元リンクは、認証が必要なAPI endpointではなく、ユーザーがブラウザで目検できる公開ページ・統計表ページを優先する。APIしか参照手段がない場合も、APIキー、アクセストークン、署名付きURLなどの秘匿情報はリンクやHTMLに含めない。
- e-Stat の政府統計コード（例: `00500222`）や統計表IDを画面に表示する場合は、可能な限りそのコードから e-Stat の公開データソースページへ遷移できるリンクを付ける。
- 絞り込み条件や分類コードが値の意味を左右する場合は、統計表ID、地域コード、カテゴリコード、検索条件などを画面上で確認できるようにする。
- 未取得、取得失敗、未実装、推計値、手入力値は混同せず、画面上でも明確に区別する。MVPで未対応の指標は「未実装（TODO）」など、未取得とは別の表示にする。
- 複数のデータ鮮度が混在する場合は、画面全体の最新取得日時だけでなく、指標ごとのローカルDB保存日時・対象期間も確認できるようにする。
