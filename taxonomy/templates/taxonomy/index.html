分類体系の入れ子構造

生物分類は、界の中に門、門の中に綱、綱の中に科・属・種が入る階層構造です。 たとえば霊長類の下にヒトが紐づくように、個別の生き物は上位分類の中へ段階的に収まります。

Taxonomy は、この portfolio の中でも技術的な側面を持つアプリです。 分類体系そのものがマトリョーシカのような入れ子構造なので、実装でも小さな Value Object を内側に持ち、 それをさらに大きな Value Object が包む形で表現します。

    +-- source / target は TaxonomyHierarchy の親子関係を指す

nodes は分類名そのものを表す点で、edges は点と点をつなぐ親子関係の線です。 たとえば「動物界」「脊索動物門」「鳥綱」はそれぞれ node になり、 「動物界から脊索動物門へ」「脊索動物門から鳥綱へ」というつながりが edge になります。 

node: 動物界
  |
  | edge: 動物界 -> 脊索動物門
  v
node: 脊索動物門
  |
  | edge: 脊索動物門 -> 鳥綱
  v
node: 鳥綱

id と name をばらばらの値として運ぶのではなく、TaxonomyHierarchyItem として1つの意味を持つ値にまとめます。 その item の並びを TaxonomyHierarchy として扱い、さらに nodes/edges 形式へ変換したものを TaxonomyGraph として扱います。 鶏の飼養分布や観察記録は、鶏の観察グラフページに分けて確認できます。
